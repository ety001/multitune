# 多音盒 MultiTune 开发计划

> 本文件根据 PRD、API 设计文档及当前代码状态整理，用于指导后续迭代。

---

## 一、当前已完成（Merged）

### 后端
- Go + Gin 骨架，SQLite (`modernc.org/sqlite`)，CGO-free 静态编译
- 身份、歌单、歌曲、扫描、音频流、播放状态、设备日志等全部 API
- 数据库迁移脚本（001/002/003）
- 路径安全校验、Range 音频流、500 错误日志等基础保障
- repo + handler 层测试覆盖

### 现代版前端（`frontend/` → `web/full/`）
- Vue 3 + Vite + Pinia + Vue Router
- 身份管理、歌单管理、文件浏览器、播放器四个页面
- App.vue 顶部导航 + 全局 mini 播放条
- 播放状态自动保存、音量记忆、播放模式切换

### 车机版前端（`web/car/`）
- 身份选择页、歌单选择页、播放器页
- ES5 + jQuery，兼容 Chrome/WebView ≤ 74
- 播放控制、播放模式、进度条、播放列表、音量
- 设备日志查看页（`logs.html`）

### 入口与静态资源
- `web/index.html` 浏览器检测 + 自动跳转车机版/完整版
- `/logo.png`、`/favicon.png`

## 二、待完成任务

按优先级和依赖关系分为 5 个阶段：

### 阶段 1：打包与本地部署（推荐下一步）

目标：让应用可以通过 Docker 一键跑起来，脱离 `go run` 和手工挂载。

- [ ] 编写根目录 `Dockerfile` 多阶段构建
  - Stage 1：pnpm 构建现代版前端（输出 `web/full/`）
  - Stage 2：Go 静态编译后端二进制
  - Stage 3：alpine 运行时，安装 ffmpeg，合并前后端静态资源
- [ ] 提供 `docker-compose.yml` 示例，方便本地验证
  - 挂载本地 `data/`、`media/`、`web/` 到容器
  - 暴露 8080 端口
- [ ] 验证容器内音频扫描、播放、文件浏览器均正常
- [ ] 更新 README.md 增加「Docker 运行」章节

### 阶段 2：车机版体验补齐

目标：让车机端成为真正可用的驾驶场景播放器。

- [ ] 车机版首页默认身份自动进入（可选开关，减少一步操作）
- [ ] 车机版播放器加入键盘事件（空格播放/暂停、左右方向键切歌）
- [ ] 车机版播放列表支持展开后点击切歌（已支持，需验证老浏览器）
- [ ] 车机版文件浏览器精简版（按文件夹添加歌曲，P1）
- [ ] 车机版音量按身份记忆（localStorage，P1，需权衡老浏览器兼容性）

### 阶段 3：现代版体验优化

目标：让 PC/大屏管理更高效。

- [ ] 文件浏览器左侧存储源树 + 右侧已选歌曲面板
- [ ] 歌单内歌曲拖拽排序（或上下移动按钮）
- [ ] 文件浏览器/歌曲库分页加载与搜索优化
- [ ] 全局加载状态、错误提示统一处理
- [ ] 播放器全屏/迷你模式切换

### 阶段 4：后端健壮性增强

- [ ] 扫描大目录时改为异步任务 + 任务状态查询（当前为同步阻塞）
- [ ] 歌曲内嵌封面提取与 `/api/songs/:id/cover` 接口
- [ ] 音频流在 USB/SMB 断开时的友好降级
- [ ] 数据库备份脚本（`song_progress` 和 `playback_states` 定期导出）
- [ ] 身份/歌单数量上限可配置化检查已做，可补充前端提示

### 阶段 5：V1.1 扩展功能

- [ ] `recent_plays` 表与最近播放记录
- [ ] 最近播放歌单自动生成
- [ ] 驾驶模式全屏界面
- [ ] 方向盘按键/快捷键绑定
- [ ] 语音控制接口预留

## 四、提交与 PR 规范

详见 `AGENTS.md`，关键提醒：

1. 提交前运行 `go vet ./...`、`gofmt -l .`、`go test -race -count=1 ./...`
2. PR Body 必须使用 `--body-file` 写入，禁止内联 `--body "..."` 中写 `\n`
3. 车机版 `web/car/` 代码保持 ES5 + 无 Flexbox/Grid
4. 每个 PR 聚焦一个功能，避免大杂烩

---

## 五、下一步建议

推荐先做 **阶段 1：打包与本地部署**。原因：

- 后端、现代版、车机版都已可独立运行，下一步需要把它们组装成可交付的容器镜像
- Docker 化后可以在本地模拟 LPK 运行环境，便于后续车机版真机测试
- 不依赖外部 LPK 配置，自主权高

如想继续推进功能体验，也可选择 **阶段 2：车机版体验补齐** 或 **阶段 3：现代版体验优化**。
